# Firestore Security Rules 検証チェックリスト

フェーズ9では次の観点を確認する。

## 許可される操作

- ログイン済みユーザーが自分の `users/{uid}` を作成・更新・読み取りできる。
- ログイン済みユーザーがグループを作成し、同時に自分の `groupMembers/{groupId_uid}` を作成できる。
- グループメンバーが同じグループの `groups`、`players`、`matches`、`hands`、`playerStats` を読み取れる。
- グループメンバーが同じグループのプレイヤー、半荘、局結果、成績を作成・更新できる。
- グループメンバーが未使用プレイヤー削除時に `players` と `playerStats` を削除できる。

## 拒否される操作

- 未ログインユーザーによるすべての読み書き。
- 別ユーザーの `users/{uid}` の読み書き。
- グループ外ユーザーによる対象グループデータの読み書き。
- `groupId`、`playerId`、`matchId`、`handId`、作成者、作成日時などの不変フィールド変更。
- `groups`、`groupMembers`、`matches`、`hands` の削除。
- 想定外フィールドを含む作成・更新。

## 補足

Rulesはクライアント側の入力検証を置き換えるものではない。点数計算、未使用プレイヤー判定、半荘進行などの業務ルールはアプリ側で検証し、Rulesでは認証・所属・不変フィールド・最低限のデータ形状を保証する。
