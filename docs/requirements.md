# 麻雀成績管理アプリ 要件定義書

## 1. 概要

本アプリは、仲間内の麻雀グループで半荘ごとの成績とプレイヤー別の通算成績を管理するWebアプリである。

1局ごとにツモ、ロン、流局、罰符などによる点数変動を入力し、半荘終了時に順位、最終持ち点、精算ポイントを算出する。また、局ごとの記録をもとに、各プレイヤーの和了率、放銃率、平均順位、トップ率などの一般的な麻雀成績を集計し、グループメンバーが閲覧できるようにする。

## 2. 目的

- 対局中にスマートフォンから素早く局結果を入力できるようにする。
- 半荘終了後の点数計算、順位計算、精算ポイント計算を自動化する。
- プレイヤーごとの長期的な成績を可視化し、仲間内で共有できるようにする。
- 将来的な詳細分析やCSV出力に拡張しやすいデータ構造で管理する。

## 3. 対象ユーザー

- 仲間内で定期的に麻雀を打つグループ。
- 対局結果をスマートフォンで入力したいメンバー。
- 自分や他メンバーの成績をランキングや個人詳細で確認したいメンバー。

## 4. 技術スタック

- フロントエンド: Next.js
- 言語: TypeScript
  - ユーザー希望のJavaScriptを尊重しつつ、データ構造の安全性を高めるためTypeScriptを採用する。
- 認証: Firebase Authentication
- データベース: Cloud Firestore
- デプロイ: Vercel
- v1では Firebase Storage と Cloud Functions は必須対象外とする。

## 5. v1の対象範囲

### 5.1 対象とする麻雀ルール

- 4人麻雀
- 東南戦
- 半荘単位の成績管理
- グループごとの既定ルール設定
- 半荘作成時のルール上書き

### 5.2 初期ルール既定値

- 開始点: 25000点
- 返し点: 30000点
- ウマ: 10-30
- トビ終了: あり
- 同点順位: 起家に近いプレイヤーを上位とする

### 5.3 v1で扱う局種別

- 和了
  - ツモ
  - ロン
- 流局
  - 各プレイヤーの聴牌、ノーテンを入力する
- 簡易罰符
  - チョンボ、誤ロンなどの特殊な点数移動を手入力できるようにする

### 5.4 v1では対象外とする機能

- 3人麻雀対応
- 東風戦対応
- 詳細牌譜入力
- 役、ドラ、巡目、鳴き、待ち形などの詳細分析
- CSV出力
- プロフィール画像管理
- 詳細な変更差分履歴
- Cloud Functionsによるサーバー側集計

## 6. 機能要件

### 6.1 認証

- Firebase Authentication の Googleログインを使用する。
- ログイン済みユーザーだけがアプリを利用できる。
- ユーザーは複数のグループに参加できる。

### 6.2 グループ管理

- ユーザーはグループを作成できる。
- グループには複数のメンバーを登録できる。
- グループごとに既定の麻雀ルールを保持する。
- グループメンバーは同じグループ内の対局、成績、ランキングを閲覧できる。

### 6.3 メンバー管理

- グループ内にプレイヤー名を登録できる。
- プレイヤーはFirebaseのログインユーザーと紐づけ可能とする。
- 紐づいていないプレイヤーも、手入力用の名前として登録できる。
- プレイヤー名は後から変更できる。
- 成績はプレイヤー単位で集計する。

### 6.4 半荘管理

- グループ内で半荘を新規作成できる。
- 半荘作成時に4人の参加プレイヤーを選択する。
- 半荘ごとに以下を保持する。
  - 対局日
  - 参加プレイヤー
  - 起家
  - 適用ルール
  - 現在局
  - 供託本数
  - 積み棒
  - 状態
- 半荘の状態は以下とする。
  - 入力中
  - 終了
  - キャンセル

### 6.5 局結果入力

局ごとに以下を入力できる。

- 局番号
  - 東1局、東2局、東3局、東4局、南1局、南2局、南3局、南4局
- 本場数
- 供託本数
- 局種別
  - 和了
  - 流局
  - 罰符
- リーチ者
- 点数変動
- 作成者
- 更新者
- 作成日時
- 更新日時

和了の場合は以下も入力する。

- 和了種別
  - ツモ
  - ロン
- 和了者
- 放銃者
  - ツモの場合はなし
- 各プレイヤーの点数増減

流局の場合は以下も入力する。

- 各プレイヤーの聴牌、ノーテン
- 各プレイヤーの点数増減

罰符の場合は以下も入力する。

- 対象プレイヤー
- 理由メモ
- 各プレイヤーの点数増減

### 6.6 点数管理

- 半荘開始時の各プレイヤーの持ち点は25000点とする。
- 各局の点数増減を保存する。
- 半荘内の現在持ち点は、開始点と局ごとの点数増減から算出する。
- 点数増減の合計は原則0になるように入力チェックする。
- 供託や積み棒を含む場合も、半荘全体で点数整合性が取れるように管理する。
- トビ終了ありの場合、いずれかのプレイヤーが0点未満になった時点で半荘を終了可能にする。

### 6.7 半荘結果

半荘終了時に以下を表示する。

- 各プレイヤーの最終持ち点
- 順位
- 素点
- ウマ
- オカ
- 合計ポイント
- トップ者
- ラス者

同点が発生した場合は、起家に近いプレイヤーを上位とする。

### 6.8 プレイヤー別成績

プレイヤーごとに以下の成績を集計する。

- 半荘数
- 参加局数
- 合計ポイント
- 平均ポイント
- 平均順位
- 平均得点
- トップ回数
- 2着回数
- 3着回数
- ラス回数
- トップ率
- 連対率
- ラス率
- 和了率
- 放銃率
- 立直和了率
- ツモ率

ユーザーの言う「放ラ率」は「和了率」を指す用語として扱う。アプリ内の表示名は、誤解を避けるため「和了率」を基本とする。

### 6.9 ランキング

- グループ内のプレイヤー成績をランキング形式で表示する。
- 初期表示では合計ポイント順とする。
- 以下の項目で並び替えできる。
  - 合計ポイント
  - 平均順位
  - 平均ポイント
  - 和了率
  - 放銃率
  - トップ率
  - ラス率

### 6.10 編集権限

- グループメンバー全員が、同じグループ内の半荘、局結果、プレイヤー情報を作成、編集、閲覧できる。
- 各データには作成者、更新者、作成日時、更新日時を保存する。
- v1では変更前後の詳細差分履歴は保持しない。

## 7. 画面要件

### 7.1 共通方針

- スマートフォンでの対局中入力を優先する。
- PCでも閲覧、編集できるレスポンシブUIとする。
- 入力中に現在点数、供託、本場、次局を確認しやすくする。

### 7.2 主要画面

#### ログイン画面

- Googleログインボタンを表示する。
- 未ログインユーザーはログイン画面に誘導する。

#### グループ一覧画面

- 参加中のグループ一覧を表示する。
- 新規グループを作成できる。
- グループを選択してグループホームへ移動できる。

#### グループホーム画面

- 最近の半荘一覧を表示する。
- ランキングの概要を表示する。
- 半荘作成、メンバー管理、成績詳細へ移動できる。

#### メンバー管理画面

- プレイヤー一覧を表示する。
- プレイヤーを追加、編集できる。
- Firebaseユーザーとの紐づけ状態を確認できる。

#### 半荘作成画面

- 参加プレイヤー4人を選択する。
- 起家を選択する。
- グループ既定ルールを確認し、必要に応じて上書きする。

#### 局入力画面

- 現在の局、本場、供託、各プレイヤーの持ち点を表示する。
- 和了、流局、罰符の入力フォームを切り替えられる。
- 保存後、次局へ進む。
- 入力済みの局結果を修正できる。

#### 半荘結果画面

- 最終持ち点、順位、ポイントを表示する。
- 半荘を終了確定できる。
- 入力済みの局履歴を確認できる。

#### ランキング画面

- グループ内の成績ランキングを表示する。
- 指標ごとに並び替えできる。

#### 個人成績詳細画面

- プレイヤーの通算成績を表示する。
- 半荘履歴を表示する。
- 和了率、放銃率、トップ率などの主要指標を表示する。

## 8. 成績指標の定義

### 8.1 共通定義

- 参加局数: 対象プレイヤーが参加した局の数。
- 半荘数: 対象プレイヤーが参加し、終了した半荘の数。
- 和了局数: 対象プレイヤーが和了した局の数。
- 放銃局数: 対象プレイヤーがロン和了に対して放銃者になった局の数。
- ツモ和了局数: 対象プレイヤーがツモで和了した局の数。
- ロン和了局数: 対象プレイヤーがロンで和了した局の数。

### 8.2 指標計算

- 和了率 = 和了局数 ÷ 参加局数
- 放銃率 = 放銃局数 ÷ 参加局数
- ツモ率 = ツモ和了回数 ÷ 総和了回数
- ロン率 = ロン和了回数 ÷ 総和了回数
- 立直和了率 = 立直して和了した回数 ÷ 立直した回数
- トップ率 = 1着回数 ÷ 半荘数
- 連対率 = 1着回数と2着回数の合計 ÷ 半荘数
- ラス率 = 4着回数 ÷ 半荘数
- 平均順位 = 順位合計 ÷ 半荘数
- 平均ポイント = 合計ポイント ÷ 半荘数

### 8.3 集計方式

- 局結果または半荘結果を保存、修正したタイミングで集計値を更新する。
- ランキング表示では保存済みの集計値を利用する。
- 将来的には集計不整合に備え、再集計処理を追加できる設計にする。

## 10. 非機能要件

### 10.1 操作性

- スマートフォンで片手入力しやすい画面構成にする。
- 対局中に現在点数がすぐ確認できるようにする。
- 局結果保存後、次の入力へ自然に進めるようにする。

### 10.2 性能

- ランキング表示では保存済み集計値を利用し、毎回全局データを再計算しない。
- グループ単位でクエリできるデータ構造にする。

### 10.3 セキュリティ

- Firestore Security Rulesで、グループメンバー以外がグループ内データを閲覧、編集できないようにする。
- ログインしていないユーザーはアプリデータにアクセスできないようにする。

### 10.4 保守性

- ルール設定、半荘、局結果、集計値を分離して管理する。
- 将来的な東風戦、3人麻雀、CSV出力、詳細分析の追加に備える。
