# 麻雀成績管理アプリ 要件定義書

## 1. 概要

本アプリは、仲間内の4人麻雀グループで半荘ごとの対局、局結果、最終結果、プレイヤー別成績を管理するWebアプリである。

Googleログイン後、ユーザーはグループを作成または既存グループへ参加し、グループ内のプレイヤー、半荘、局履歴、半荘結果を記録・閲覧できる。

## 2. 目的

- スマートフォンから対局中に素早く局結果を入力できるようにする。
- 和了、流局、罰符による点数変動を記録し、現在持ち点を自動計算する。
- 半荘終了時に順位、素点、ウマ、オカ、合計ポイントを自動計算する。
- グループ内メンバーが同じデータを共有できるようにする。
- 将来的にランキング、個人成績詳細、CSV出力へ拡張しやすいデータ構造にする。

## 3. 対象ユーザー

- 仲間内で定期的に4人麻雀を行うグループ。
- 対局結果をスマートフォンで記録したいメンバー。
- 自分や他メンバーの成績を確認したいメンバー。

## 4. 技術スタック

- フロントエンド: Next.js
- 言語: TypeScript
- 認証: Firebase Authentication / Googleログイン
- データベース: Cloud Firestore
- デプロイ: Vercel

v1では Firebase Storage、Cloud Functions は対象外とする。

## 5. 対象ルール

### 5.1 対象範囲

- 4人麻雀
- 東南戦
- 半荘単位の成績管理
- グループごとのルール設定
- 半荘作成時のルール上書き

### 5.2 初期ルール既定値

- 開始点: 25000点
- 返し点: 30000点
- ウマ: +30 / +10 / -10 / -30
- トビ終了: あり
- 同点順位: 起家に近いプレイヤーを上位
- 連荘ルール: 親和了・親テンパイ流局で連荘

### 5.3 半荘作成時に設定できるルール

- 開始点
- 返し点
- ウマ
- トビ終了
  - 飛びあり: 誰かがマイナスになったら半荘終了
  - 飛びなし: 誰かがマイナスになっても続行
- 連荘ルール
  - 親和了・親テンパイ流局で連荘
  - 親和了のみ連荘
  - 流局は親テンパイに関係なく連荘

## 6. 機能要件

### 6.1 認証

- Googleログインを使用する。
- 未ログインユーザーはアプリの主要機能を利用できない。
- ログインユーザーの `uid`、`displayName`、`email`、`photoURL` を取得・保存できる。

### 6.2 グループ管理

- ログインユーザーはグループを作成できる。
- グループ作成者は自動的にグループメンバーとして登録される。
- グループには既定ルールを保存する。
- グループIDを表示し、他ユーザーへ共有できる。
- ログイン済みユーザーは、共有されたグループIDを入力して既存グループに参加できる。
- 参加後は、自分が参加しているグループ一覧に対象グループが表示される。

### 6.3 プレイヤー管理

- グループ内にプレイヤーを登録できる。
- プレイヤー名を編集できる。
- Firebaseユーザーと紐づかない手入力プレイヤーを作成できる。
- Firebaseユーザーとの紐づき状態を確認できる。
- 未使用プレイヤーは削除できる。
- 対局や局履歴で使用済みのプレイヤーは、成績整合性を壊さないよう削除制限を設ける。

### 6.4 半荘作成

- グループ内のプレイヤーから4人を選択して半荘を開始できる。
- 東家、南家、西家、北家の初期席を指定する。
- 起家は東家として扱うため、起家の別選択は不要とする。
- 半荘作成時に、グループ既定ルールを確認・上書きできる。
- 半荘開始時の各プレイヤー持ち点は開始点とする。
- 半荘作成後、局入力画面へ移動できる。

### 6.5 現在局・席表示

- 現在局は東1局から南4局まで管理する。
- 半荘開始時は東家が親である。
- 次局に進むたびに、現在の東家・南家・西家・北家を自動で切り替える。
- UI上で現在の家を各プレイヤーに表示する。
- 現在の親は `親` ラベルと強調表示で分かりやすくする。
- 和了者、放銃者、聴牌者、リーチ者、点数増減入力でも現在の家を表示する。

### 6.6 局結果入力

局ごとに以下を保存できる。

- 局
- 本場
- 供託本数
- 局種別
  - 和了
  - 流局
  - 罰符
- リーチ者
- 点数増減
- 作成者
- 更新者
- 作成日時
- 更新日時

### 6.7 和了入力

- 和了種別としてロン・ツモを選択できる。
- 和了者を選択できる。
- ロンの場合、放銃者を選択し、ロン支払い点を入力する。
- ツモの場合、和了者が親か子かで入力方法を切り替える。
  - 親ツモ: 各自支払い点を1つ入力する。
  - 子ツモ: 親支払い点、子支払い点を入力する。
- 現在供託と今回リーチ棒は、すべて和了者が回収する。
- 和了時の点数増減は自動計算する。
- 点数増減合計は、供託回収分と整合することを確認する。

### 6.8 流局入力

- 聴牌者を選択できる。
- リーチ者を選択できる。
- ノーテン罰符を自動計算する。
- 流局時のリーチ棒は供託として次局へ持ち越す。
- 流局後の供託本数をUIで確認してから保存する。
- 流局時は、リーチ棒が供託に移るため、点数増減合計が0にならない場合を許容する。

### 6.9 局進行・連荘

- 親が和了した場合は、同じ局のまま本場を+1する。
- 子が和了した場合は、次局へ進み、本場を0に戻す。
- 流局時は本場を+1する。
- 流局時に親が続行するかどうかは、半荘作成時の連荘ルールに従う。
- 流局で親が連荘しない場合は、次局へ進み、本場のみ+1する。
- 供託は和了時に0へ戻り、流局時は現在供託 + 今回リーチ本数になる。

### 6.10 トビ終了

- 半荘ルールで「飛びあり」を選んだ場合、局保存後に誰かの持ち点がマイナスになった時点で半荘を終了する。
- 半荘ルールで「飛びなし」を選んだ場合、誰かがマイナスになっても半荘を続行する。
- トビ終了時は、局結果保存と同時に半荘結果を確定保存する。

### 6.11 半荘結果

- 現在の全局履歴から最終持ち点を計算する。
- 最終持ち点から順位を計算する。
- 同点の場合は起家に近いプレイヤーを上位とする。
- 以下を計算・表示する。
  - 最終持ち点
  - 順位
  - 素点
  - ウマ
  - オカ
  - 合計ポイント
- 半荘を手動終了できる。
- 半荘終了後は `matches.status` を `finished` にし、`finalResults` を保存する。
- 終了済み半荘を選択した場合、局入力ではなく半荘結果を表示する。

### 6.12 成績・ランキング

以下は実装対象だが、v1の後続フェーズで強化する。

- 局結果保存時、修正時、半荘終了時に `playerStats` を更新する。
- グループ内ランキングを表示する。
- プレイヤー別成績詳細を表示する。
- ランキングは以下の指標で並び替えできる。
  - 合計ポイント
  - 平均順位
  - 平均ポイント
  - 和了率
  - 放銃率
  - トップ率
  - ラス率

## 8. セキュリティ要件

- 未ログインユーザーはアプリデータにアクセスできない。
- ログイン済みユーザーは、自分の `users/{uid}` のみ作成・更新できる。
- グループメンバーは同じグループ内のデータを閲覧・編集できる。
- グループ未参加ユーザーは、グループID指定による存在確認以外のグループ内データを閲覧・編集できない。
- ログイン済みユーザーは、存在するグループに対して自分自身の `groupMembers` ドキュメントだけ作成できる。
- 他人を勝手にグループへ参加させることはできない。
- `matches`、`hands`、`players`、`playerStats` は対象グループのメンバーだけが操作できる。

## 9. 非機能要件

### 9.1 操作性

- 対局中のスマートフォン入力を優先する。
- 局入力画面では、現在局、本場、供託、現在持ち点、現在の親を確認しやすくする。
- ロン・ツモ・流局の入力は、必要最小限の項目で点数計算できるようにする。
- 入力後は自然に次の局状態へ進む。

### 9.2 保守性

- ルール設定、半荘、局結果、集計値は分離して管理する。
- 将来的な東風戦、3人麻雀、CSV出力、詳細分析に備えたデータ構造にする。
- 既存データに新しい任意項目が存在しない場合でも、既定値で動作する。

### 9.3 性能

- ランキング表示では保存済み集計値を利用し、毎回全局データを再計算しない設計を目指す。
- グループ単位でクエリ可能なデータ構造にする。

## 10. v1対象外

- 3人麻雀
- 東風戦
- 詳細な役入力
- 符、翻、点数表からの詳細計算
- リーチ後の待ち、役、ドラ、巡目などの分析
- CSVエクスポート
- グラフ表示
- プロフィール画像管理
- 公開ランキングURL
- 半荘結果の共有画像生成
- Cloud Functionsによるサーバー側集計

## 11. 受け入れ条件

- Googleログインできる。
- グループを作成できる。
- グループIDで他ユーザーが既存グループに参加できる。
- グループ内にプレイヤーを登録できる。
- 4人を選択して半荘を開始できる。
- 半荘作成時にトビ終了と連荘ルールを選択できる。
- 局ごとに和了、流局、罰符の結果を保存できる。
- ロン、ツモ、流局の点数増減を自動計算できる。
- リーチ棒と供託を正しく扱える。
- 親和了、流局、連荘ルールに応じて次局と本場が正しく更新される。
- 現在の家と親がUI上で分かりやすく表示される。
- 飛びありの場合、誰かがマイナスになったら半荘が終了する。
- 飛びなしの場合、誰かがマイナスになっても半荘が継続する。
- 半荘終了時に順位、最終持ち点、素点、ウマ、オカ、合計ポイントを確認できる。
- グループメンバーだけが対象グループのデータを閲覧・編集できる。
